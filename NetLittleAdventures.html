Little Adventures

Team Ninja Turtles
Lucy Murphy, Francesca Nemati, Philip Von Mueffling

"Abstracted" description of project:

Our model now follows just one path, but incorporates elements from previous path ideas. Users now will select which character they want to use for their journey, and that choice will be consistent throughout the game. Once a character is selected, the user plays as that character to complete the tasks. The game starts with a home screen, in which the user can move right to go to the next area of the map, the playground. In the playground, there is a treehouse the user can enter, where they will obtain a ladder by knocking it down with a hammer they find. Once they get this ladder, they can continue with the journey and climb up a tree to talk with a penguin, which gives the user a task. The user will then continue their journey in the part of the map that is to the left of the homescreen. This is the entrance to a town, where there is a hardware shop and a ducky shop. In the town, the penguin will also be there so that the user can interact with him. In the hardware shop, the user finds a coin from a gumball machine, then uses it to buy an axe. They use this axe to chop down part of the fence and obtain a twig, which the penguin wanted for his nest. When the twig is presented to the penguin, the user then continues with the last task: completing a maze. The penguin needs the user to complete the maze and return his hat in exchange for letting the user borrow his plane. This plane is the ultimate goal, since it will be used to fly home. At the end of the maze, the user can see his character flying home in the plane, and returning home to the characterâ€™s mom. At this point, the game is complete.

Reflection

Procedural Abstraction:
I used proceduaral abstraction by creating several helper functions. One of these helper functions was the make_square() function. This function made a filled in square in what ever color was called with the function. This was used to set up the obsticals so it was easier to see the path. Using this function made the code more robust because I did not need to repeat the same lines of code over and over. I also made helper functions for each individual paths driving instructions. This way I can call the specific functions in one larger main function after the set up grid has been executed.

How Could This Be More Efficient?:
This code is not exactly efficient as it is all hard coded, and therefor is only partially reusable. In order to make this code more efficient I would use sensors so the robot knows when it is in contact with an obstical. This way the only information it would need is any 2D list.
